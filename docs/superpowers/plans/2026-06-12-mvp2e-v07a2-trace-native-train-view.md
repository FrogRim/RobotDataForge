# MVP-2E v0.7a.2 Trace-Native Train View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` or
> `superpowers:subagent-driven-development` to implement this plan task-by-task.
> This is a planning artifact only. Do not execute implementation until the user

**Goal:** Implement `v0_7a_2`, a child policy slice that builds candidate and baseline
training views directly from full-horizon actual Isaac runtime trace rows, preserving
env-native HOLD authority and keeping held-out `21000-21049` sealed.

**Source spec:**
`docs/superpowers/specs/2026-06-12-mvp2e-v07a2-trace-native-train-view-design.md`

**Current blocker:** `v0_7a_1` correctly hydrates parent HDF5 rows from runtime traces,
but the authenticated HDF5 row window does not include the seated/HOLD trace tail:

```text
candidate_phase_row_counts: ALIGN=1280, DESCEND=0, HOLD=0
offline_train_fit_gate.passed=false
failure_reason=required_phase_missing
```

`v0_7a_2` fixes the train-view source, not the success metric.

---

## RALPLAN-DR Summary

### Principles

1. Env-native success mask remains the only HOLD/seating authority.
2. Trace-native rows must be authenticated from parent `train_generation_runtime_gate`.
3. Candidate and baseline must share the same feature schema, trainer, action schema,
   and behavior phase rule.
4. Held-out `21000-21049` and calibration remain sealed until explicit downstream gates pass.
5. Failure of trace-native offline fit or Phase E is evidence for `v0_7b`, not a reason
   to relax thresholds.

### Decision Drivers

1. Current blocker is training-view loss of HOLD rows, not proven policy-class failure.
2. MVP-2 claim integrity requires baseline/candidate artifacts from the same evidence family.
3. The next step should be offline-first and avoid unnecessary Isaac runtime until the
   train-view gate is proven.

### Viable Options

**Option A: Trace-native full-horizon train view. Chosen.**

Pros:
- Directly fixes the observed HDF5 window loss.
- Uses actual Isaac runtime traces already generated by the v0.6 train gate.
- Can create candidate and baseline policies from the same source family.
- Preserves phase-conditioned NumPy BC before escalating to a stronger policy class.

Cons:
- Requires strict trace row validation and action normalization.
- Baseline uncurated policy may still be noisy or weak.

**Option B: Jump to `v0_7b` residual servo BC. Deferred.**

Pros:
- Likely more expressive if phase-conditioned BC cannot learn the behavior.
- Already identified as the fallback path.

Cons:
- Changes policy class before fixing the proven train-view source bug.
- Would leave an unanswered question: whether current BC would pass with full-horizon rows.

**Option C: Relax HOLD or depth thresholds. Rejected.**

Pros:
- Minimal code change.

Cons:
- Reintroduces geometry proxy authority after env-native authority was fixed.
- Weakens proof integrity and conflicts with the spec.

### ADR

**Decision:** Add `v0_7a_2` as a trace-native child slice. Candidate view uses
`generated_success_trace_paths`; baseline view uses all `generated_trace_paths`.

**Drivers:** repair the actual train-view blocker, preserve claim integrity, and keep
the policy class unchanged until it is honestly tested.

**Alternatives considered:** direct `v0_7b` residual servo BC and threshold relaxation.

**Why chosen:** the evidence shows HOLD rows exist in runtime traces but not in the
parent HDF5 train view. The smallest valid repair is to train from the trace rows.

**Consequences:** `v0_7a_2` may still fail offline fit or Phase E. If it does, the
next valid step is a `v0_7b` residual servo BC spec.

**Follow-ups:** passing offline fit allows Phase E expressibility only. Passing Phase E
allows a later calibration presignal plan; it does not open held-out automatically.


Add the same behavior phase rule constant to
`scripts/run_mvp2b_isaac_proof_evaluator.py` near
`V07A1_BEHAVIOR_PHASE_RULE_VERSION`.

### Trace-Native Source Contract

Do not use parent HDF5 rows as the primary row source.

```text
candidate rows = all valid rows from train_generation_runtime_gate.generated_success_trace_paths
baseline rows  = all valid rows from train_generation_runtime_gate.generated_trace_paths
```

Each trace path must be in the parent gate payload and must be hashed into the manifest.

### Behavior Phase Rule

Add a v0.7a.2 rule equivalent to v0.7a.1:

```text
HOLD    = env_native_success_mask == true
DESCEND = not HOLD AND lateral_error_m <= 0.001
ALIGN   = not HOLD AND lateral_error_m > 0.001
```

Runtime prediction must use this rule only when policy artifact
`behavior_phase_rule_version == "env_native_hold_v0_7a_2"`.

## Acceptance Criteria

- `v0_7a_2` config and manifest exist.
- Candidate view is built from `generated_success_trace_paths` full trace rows.
- Baseline view is built from `generated_trace_paths` full trace rows.
- Candidate view contains `HOLD > 0`.
- Baseline noise/failure mix is derived from parent gate counts.
- Candidate and baseline policy artifacts are written when their rows are valid.
- Runtime evaluator uses `env_native_hold_v0_7a_2` only for v0.7a.2 artifacts.
- Expressibility guard blocks before Isaac when offline gate is missing/failed.
- Held-out `21000-21049` remains unopened.
- Existing v0.7a and v0.7a.1 tests remain passing.
- No threshold relaxation or residual servo BC is introduced.

---

## Stop Conditions

Stop and report if:

- `train_generation_runtime_gate.json` is missing or `passed=false`.
- Success trace count is below 20.
- Runtime trace rows do not contain env-native mask fields.
- Runtime trace rows do not contain `normalized_action`.
- `normalized_action` cannot be mapped to `ACTION_SCHEMA` without inventing semantics.
- Candidate trace-native view still has `HOLD=0`.
- Candidate offline fit gate fails.
- Phase E candidate expressibility is below `2/5`.
- Baseline policy cannot be written from `generated_trace_paths`.
- Implementation would require opening held-out `21000-21049`.
- Implementation would require calibration before Phase E.
- Implementation would require changing env-native authority, policy class, or HOLD thresholds.

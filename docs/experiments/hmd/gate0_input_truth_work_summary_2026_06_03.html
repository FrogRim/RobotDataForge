무엇을 바꿨나

classification은 허용 권한을 갖지 않는다. 오직 unsafe/invalid 입력을 설명하고, 이미 존재하는 hold/resume/recenter 계열에 block-only metadata를 추가한다.

파일 변경 요약 의미
scripts/rdf_input_sources.py InputTruthClassification, classify_input_truth, merge_input_truth_soft_blocks, input_truth_control_state 추가 frame별 입력 truth를 invalid/unsafe/valid로 설명하고 block-only control state를 생성한다.

scripts/rdf_isaac_runtime_recorder.py tracking-gate hold 프레임에 input-truth classification/block metadata 연결 live recorder artifact에서도 held frame이 왜 학습/action authority에 부적합한지 남긴다.
apps/api/tests/test_teleop_diagnostics_scripts.py classification priority, provenance-only valid, block-only merge, epoch summary, recorder 연결 테스트 추가 classification이 권한을 강제로 허용하지 않는다는 계약을 fixture로 고정한다.

보존한 경계

이번 구현은 Gate 0 판정 기준을 완화하지 않았고, axis/gain/recenter 대수술도 하지 않았다. task success와 data quality 분리 원칙도 그대로 유지했다.

  • 절대 grant하지 않음

    resume_allowed, recenter_allowed, gate0_pass, gate_a_collection_allowed를 classification이 생성하거나 true로 만들지 않는다.

  • block-only authority

    invalid/unsafe classification은 action_hold, resume_block, recenter_block만 추가한다.

  • Gate 0 threshold 유지

    raw wrist jump, tracked rate, XR valid rate, tracking loss 기준은 완화하지 않았다.

  • Gate A readiness 아님

    1차 완료는 implementation acceptance다. collection readiness는 물리 HMD Gate 0 재실행으로만 판단한다.

  • 기존 audit/proof 흐름 유지

    evaluator, curator, export, replay proof 경계를 새 classification이 대체하지 않는다.

  • 원인 설명 강화

    실패한 frame과 epoch에 reason code를 남겨 다음 HMD 검증에서 raw input 문제와 control 문제를 분리할 수 있다.

MVP 완성까지 어디까지 왔나?

Robot Data Forge는 로봇이 배울 수 있는 “좋은 조작 데이터”를 만드는 공장이다. 지금은 데이터 저장과 검수 공장은 많이 완성됐고, 다음 큰 숙제는 Quest 3 손추적이 끊기지 않게 안정화해서 학습 가치까지 증명하는 것이다.

현재 상태 보기 최종 제품 그림 입력 신호 리서치 HMD 전 Step 1 남은 일 보기 검증 명령 보기

오늘의 한 줄 결론

  • 공장trajectory 저장·평가·curation·export 흐름은 동작한다.
  • 현재 막힘손추적 loss/spike 때문에 로봇이 계속 멈추거나 다시 기준점을 잡는다.
  • 금지이 상태에서는 Gate A 수집과 axis/gain 튜닝을 재개하지 않는다.

Current blocker

왜 아직 손동작을 로봇이 잘 따라오지 못하나?

최신 로그 기준으로는 “로봇 제어 코드가 아예 안 켜진 것”도 아니고, “목표 위치 계산이 계속 쓰레기값을 누적하는 것”도 아니다. 로봇은 명령이 들어오면 따라간다. 문제는 OpenXR 오른손 wrist가 중간중간 사라지거나 갑자기 튀어서, 안전장치가 로봇을 멈추는 것이다.

#30.3 No-Go 감지

현재 빨간불: handtracking loss 과다

전체 180 frame 중 손추적이 안정적인 구간이 부족했다. 그래서 controller가 held=116 frame 동안 로봇 제어를 붙잡았고, accepted=54 frame에서만 실제 명령을 보냈다.

180저장된 trajectory frames
44.4%evaluation tracking loss rate
57.8%right hand tracked rate
27회valid-to-valid wrist jump > 10cm
0.91m최대 wrist jump
H7 PASS명령이 들어오면 EEF는 방향을 따라감
중학생 버전 설명: 게임 컨트롤러가 가끔 연결이 끊기고, 캐릭터 위치가 순간이동한다고 생각하면 된다. 캐릭터가 멍청해서 못 움직이는 것이 아니라, 입력 신호가 위험하게 튀어서 안전장치가 “잠깐 멈춰!”라고 하는 상태다. 자세한 리서치와 red flag 조절안은 입력 신호 리서치 하위 페이지에 따로 정리했다.

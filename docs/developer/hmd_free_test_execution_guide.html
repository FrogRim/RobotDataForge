Developer test guide

이제 기본 테스트는 HMD가 아니라 HMD-free data trust proof다

OpenXR, ALVR, SteamVR, Quest handtracking, Isaac Sim GUI는 더 이상 현재 1차 proof의 기본 의존성이 아니다. 기본 검증은 repo-local synthetic/scripted fixture와 API/test/export 경로로 끝나야 한다. HMD와 Isaac live 경로는 실험 adapter를 다시 검증할 때만 별도로 실행한다.

Default HMD-free proof 커밋 전 기본 acceptance. OpenXR/ALVR/SteamVR 불필요.
Runtime Local Python/API tests pytest, compileall, ruff, HDF5 export, trainer smoke 중심.
Optional Isaac / HMD adapter Quest/OpenXR 경로를 다시 살릴 때만 release validation으로 실행.
Not default Gate A collection physical HMD Gate 0가 다시 통과하기 전까지 재개하지 않음.

무엇이 달라졌나

예전에는 “Quest/ALVR/SteamVR/OpenXR/Isaac Lab이 실제로 움직이는가”가 전체 프로젝트의 첫 증명처럼 취급됐다. 지금은 그 경로를 experimental adapter로 내리고, RDF core가 구매자에게 신뢰 가능한 dataset artifact를 만들 수 있는지를 먼저 검증한다.

질문 이전 HMD-first 기준 현재 data trust 기준
기본 proof에 OpenXR/ALVR/SteamVR이 필요한가? 사실상 필요했다. HMD stream stability가 병목이었다. 아니다. 기본 proof는 HMD-free fixture로 실행한다.
기본 proof에 Isaac Sim GUI가 필요한가? live teleop 확인을 위해 필요했다. 아니다. 현재 proof command는 repo-local Python 경로로 돈다.
그럼 Isaac은 버렸나? 아니다. 아니다. Isaac/HMD는 experimental adapter와 미래 physical validation 경로로 남긴다.
Gate A collection을 다시 해도 되나? HMD 안정성만 좋아 보이면 시도하고 싶었다. 아니다. physical HMD Gate 0 rerun이 통과하기 전까지 금지다.

기본 테스트 세트

RDF core나 data trust layer를 건드렸다면 아래가 현재 기준의 1차 acceptance다. 이 세트는 Quest, SteamVR, ALVR, OpenXR runtime, Isaac GUI 없이 실행되어야 한다.

Data trust proof

uv run python scripts/run_data_trust_layer_proof.py --clean --pretty

Targeted proof tests

uv run pytest apps/api/tests/test_data_trust_layer_proof_script.py -q uv run pytest apps/api/tests/test_mvp1_trainer_smoke_script.py apps/api/tests/test_mvp1_proof_audit_script.py -q

Static checks

uv run python -m compileall -q scripts apps/api/app apps/api/tests uvx ruff check scripts/run_data_trust_layer_proof.py apps/api/tests/test_data_trust_layer_proof_script.py

무엇을 바꿨는지에 따른 실행 기준

커밋 전에 모든 테스트를 무조건 돌리는 대신, 변경 범위별로 증명해야 할 claim을 분리한다.

Docs only

문서 위치, 링크, HTML만 바꾼 경우

HTML/Markdown 링크 검증과 whitespace check면 충분하다. Isaac/HMD 실행은 필요 없다.

python3 -m compileall -q scripts apps/api/app apps/api/tests # docs-only면 생략 가능 git diff --check -- index.html docs Handoff.md README.md tasks/todo.md # HTML/Markdown local-link validation은 docs portal worklog의 stdlib validator 기준으로 실행
Core proof

proof runner, evaluator, curator, export, trainer smoke를 바꾼 경우

HMD-free data trust proof와 관련 pytest를 실행한다. 이것이 현재 1차 제품 proof다.

uv run python scripts/run_data_trust_layer_proof.py --clean --pretty uv run pytest apps/api/tests/test_data_trust_layer_proof_script.py -q uv run pytest apps/api/tests/test_mvp1_trainer_smoke_script.py apps/api/tests/test_mvp1_proof_audit_script.py -q
API / services

API service나 evaluator/curator 공통 로직을 바꾼 경우

관련 targeted tests 뒤에 API 전체 테스트를 돌린다. 여전히 HMD나 Isaac은 필요 없다.

uv run pytest apps/api/tests -q uv run python -m compileall -q apps/api/app apps/api/tests scripts
Web UI

Next.js frontend를 바꾼 경우

문서 포털 HTML과 별개다. `apps/web`을 바꾸면 web lint/build를 따로 봐야 한다.

npm --prefix apps/web run lint npm --prefix apps/web run build
HMD adapter

Quest/OpenXR/HMD live adapter를 다시 건드린 경우

이때만 physical validation이 필요하다. 이것은 implementation acceptance가 아니라 release validation이다. Gate A collection은 Gate 0 통과 전까지 계속 금지다.

./scripts/run_hmd_axis_debug.sh gate0-static --no-start-xr ./scripts/run_hmd_axis_debug.sh gate0-all --no-start-xr

Isaac Sim은 언제 쓰나

현재 data trust layer proof는 Isaac Sim GUI에 의존하지 않는다. 다만 Isaac 기반 live teleop, physical HMD adapter, 또는 실제 robot/sim collection UX를 검증하는 순간에는 여전히 Isaac이 필요하다.

정리 커밋 전 기본 테스트는 Isaac 없이 끝나야 한다. Isaac/Quest/OpenXR/ALVR/SteamVR는 “RDF core proof”가 아니라 “experimental HMD adapter release validation”을 할 때만 켠다.

커밋 전에 추천하는 분리

현재 워크트리는 docs, data trust proof, API/scripts, web 변경이 섞여 있을 수 있다. 한 커밋에 모두 넣으면 테스트 기준도 전체로 커진다.

커밋 종류 넣을 내용 필수 검증
docs portal commit root/docs portal, file moves, explanation HTML HTML/Markdown link validation, `git diff --check`
data trust proof commit HMD-free proof runner, tests, generated proof docs proof command, targeted proof pytest, compileall, ruff
runtime/web commit API/scripts/web runtime behavior affected targeted tests plus broad API/web checks

관련 문서

현재 테스트 기준을 이해할 때 같이 보면 되는 문서들이다.

  • Data trust layer reset왜 HMD-first에서 HMD-free proof로 옮겼는지.
  • Project instructions현재 product boundary와 금지 claim.
  • Worklog실제 구현 및 검증 기록.
  • Proof runner현재 기본 acceptance command의 entrypoint.

1. 방향 결정

MVP-2는 단일 policy uplift gate가 아니라 staged learning proof로 진행한다.

New direction
Keep MVP-1

XR/HMD raw trajectory를 검증 가능한 learning-ready dataset artifact로 만드는 파이프라인.

Move to MVP-2 Uplift

Curated policy가 uncurated baseline보다 나은지 보는 downstream learning proof.

Do not infer Failure

현재 uplift 0.0은 전체 방향 실패가 아니라 현재 data/policy/eval 조합의 negative evidence다.

2. 왜 이렇게 잡는가

candidate_success_rate > baseline_success_rate가 항상 성립하지 않아도 방법론이 틀렸다는 뜻은 아니다.

올바른 해석

Uplift는 데이터 품질, 데이터 규모, task transition coverage, action contract, policy class, training recipe, held-out 평가 난이도가 모두 맞아야 나오는 downstream 결과다. 따라서 한 번의 zero-uplift는 "학습 증명이 아직 안 됐다"이지 "validated dataset pipeline이 무가치하다"가 아니다.

금지할 해석

Curated가 baseline보다 못 이겼다고 해서 curation gate를 느슨하게 만들거나, tiny dataset과 smoke-grade trainer 결과를 제품 실패로 과잉 일반화하면 안 된다. 다음 실험 전에 더 쉬운 sanity gate부터 닫아야 한다.

3. MVP-2 Proof Ladder

다음 held-out A/B는 아래 전제들이 통과된 뒤에만 의미가 있다.

Before next A/B

Train-set overfit sanity

강한 policy가 accepted train trajectories 자체를 재현할 수 있는지 먼저 확인한다.

실패하면 dataset/action/loader 계약 문제를 의심한다.

Replay/action contract sanity

학습에 쓰는 action이 evaluator/replay 환경에서 같은 의미로 실행되는지 확인한다.

성공처럼 보이는 trajectory라도 replay/action contract가 깨지면 학습 증거가 아니다.

Transition coverage audit

APPROACH, CONTACT, INSERT, SEAT 전이 구간이 accepted set 안에 충분히 들어 있는지 본다.

Stable SEAT/hold만 있으면 policy가 삽입 절차를 배울 근거가 부족하다.

Stronger policy/trainer

Smoke-grade linear BC를 넘어서 ACT, Diffusion Policy, LeRobot 계열 등 더 적합한 policy path를 고른다.

약한 모델 실패는 데이터 실패와 구분해야 한다.

Dataset size and coverage ablation

episode 수, scene/pose 다양성, transition 분포를 늘릴수록 train/eval 지표가 어떻게 바뀌는지 기록한다.

단순 accepted count보다 coverage가 중요하다.

Curation ablation

같은 task/split 조건에서 uncurated baseline과 curated candidate를 동일 contract로 export한다.

curation 효과만 비교하려면 나머지 변수를 고정해야 한다.

Held-out policy A/B

동일 scenario/seed/evaluator에서 candidate success rate가 baseline보다 높은지 측정한다.

양성이면 learning-proven claim 후보, 음성이면 negative report로 보존한다.

4. 외부 기준에서 배운 것

선행 프로젝트들도 curation 하나로 끝내지 않고 데이터 규모, 다양성, 표준화, 강한 policy, 평가를 분리한다.

DROID Scale & diversity

Quest 기반 teleop을 포함한 대규모 조작 dataset은 scene/task/operator 다양성과 calibration metadata를 품질 인프라로 다룬다.

droid-dataset.github.io

Open X-Embodiment Standardized data

여러 robot/task dataset을 표준화하고 모델 평가를 분리한다. RDF도 schema/export/eval contract를 먼저 고정해야 한다.

arxiv.org/abs/2310.08864

MimicGen Source demos first

소수의 신뢰 가능한 human demos를 기반으로 대규모 synthetic demos를 만든다. RDF도 source demo quality를 먼저 닫아야 한다.

mimicgen.github.io

Diffusion Policy Policy capacity

contact-rich manipulation에는 단순 linear BC보다 강한 policy class가 필요할 수 있다.

arxiv.org/abs/2303.04137

LeRobot End-to-end stack

데이터 수집, 저장, streaming, model training을 하나의 재현 가능한 stack으로 보는 방향이 RDF MVP-2와 맞다.

arxiv.org/abs/2602.22818

robomimic Controlled evaluation

데모 기반 학습은 모델/데이터/eval protocol을 분리해서 반복 측정해야 한다.

robomimic.github.io

5. 결과 판정 규칙

음성 결과를 숨기지 않고, 어떤 결론을 낼 수 있는지 엄격히 제한한다.

결과 해석 다음 조치
train-set overfit fail dataset/action/loader/policy contract 중 하나가 학습을 방해한다. held-out A/B 중지, action schema와 loader부터 수정.
transition coverage fail accepted dataset이 삽입 절차를 충분히 담지 못한다. APPROACH/CONTACT/INSERT/SEAT 전이 demo를 다시 수집.
baseline = candidate = 0 현재 policy/eval 난이도에서 둘 다 task를 못 푼다. policy capacity, data scale, task curriculum을 먼저 개선.
candidate <= baseline curation uplift는 아직 증명되지 않았다. negative report 보존, curation criterion과 dataset composition 분석.
candidate > baseline MVP-2 learning-proven claim 후보가 생긴다. confidence interval, held-out split, repeat run으로 claim 강도 확인.
